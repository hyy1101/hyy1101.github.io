LSTM for Regression Using the Window Method

我们也可以把问题想象成多维，最近的时间的步长可以用来做下一个时间步长的预测（也叫时间窗口法），窗口的大小可以作为一个参数，可以根据问题是集调整。

比如预测未来t+1，我们可以使用当天和以前几个时间的数据：t，t-1，t-2作为输入变量。
简化后，问题变为：输入变量是 t-2, t-1, t，输出变量应该是t+1。可以把前面的create_dataset()函数的look_back参数从1设置为3

LSTM for Regression with Time Steps

LSTM的参数也包括时间步长

有的序列可能有不同的时间步长。比如，你可能因为物理测量原因导致一个点测量错误，每个实例都是一个观测样本，使得事件将是时间步长，观测变量将是样本。

时间步长将会提供另外一种思考时间序列的问题。像上面的窗口示例一样，我们可以在我们的时间序列中使用前时间步长作为输入来预测下一个时间步长的输出。而不是将过去的观测作为单独的输入功能，我们可以使用它们作为时间步长的一个输入功能，这确实是一个更准确的框架的问题

我们可以这样做，使用相同的数据，在以前的窗口为基础的例子上，除了当我们重新塑造数据，我们设置列为时间步长的维度，并且将特征该回为1。比如:

# reshape input to be [samples, time steps, features]
trainX = numpy.reshape(trainX, (trainX.shape[0], trainX.shape[1], 1))

LSTM with Memory Between Batches

LSTM有记忆功能，能够记住长序列。

通常情况下，当神经网络拟合调用模型训练时，比如model.predict() 或者 model.evaluate()时，每一次training batch后都会状态复位。

当LSTM网络内部通过创建LSTM 层 “stateful”来清除keras时，我们可以更好的拟合问题。这意味着它可以在整个训练序列上建立状态，甚至可以在预测时保持状态。

这样就要求，你和模型的时候，训练集的数据不能被打乱，每次训练数据训练一个epoch后，它也需要明确的进行网络状态的复位即调用model.reset_states()函数，也就是说，我们必须创建自己的外部循环和内部循环的epoch。每个epoch调用model.fit() 和model.reset_states()。比如：

这意味着我们必须每个时代的呼叫模型在创造我们自己的时代和外环。fit() reset_states()和模型。例如:

for i in range(100):
    model.fit(trainX, trainY, nb_epoch=1, batch_size=batch_size, verbose=2, shuffle=False)
    model.reset_states()

最后，当 构建LSTM时，状态参数必须设置为true，而不是指定输入尺寸。我们必须把一些样本硬编码在一个batch中，时间步长的数量和一个时间步长的样本特征数量可以设置在batch_input_shape这个参数中。
例如：

model.add(LSTM(4, batch_input_shape=(batch_size, time_steps, features), stateful=True))

在评估模型和预测时，必须使用相同的batch size。例如:

model.predict(trainX, batch_size=batch_size)

Stacked LSTMs with Memory Between Batches

最后，我们再看一下使用lstm最大的好处：事实上，他们可以成功地训练堆叠的lstm到深层网络架构。

LSTM网络可以堆叠在keras上。同样的，其他层类型也可以被堆叠。另外的配置，需要的是一个对应层的每个后续层之前必须返回对应的序列。这可以通过设置return_sequences参数为True来完成。

比如上一部分，我们可以将stateful LSTM设置为两层。设置如下：

model.add(LSTM(4, batch_input_shape=(batch_size, look_back, 1), stateful=True, return_sequences=True))
model.add(LSTM(4, batch_input_shape=(batch_size, look_back, 1), stateful=True))
